# 2026년 4월 24일 작업 요약 (홍삼스파 ERP)

## 📌 주요 작업 내용

### 1. 보안 및 서버 접근 체계 정립
*   **플랫폼 명시:** AWS EC2가 아닌 **Amazon Lightsail** 환경임을 확인하고, `ERP 설계안.txt`에 명확히 기록.
*   **배포 룰 확립:** Root 계정(`bestsalesman@daum.net`)과 OTP 인증을 통해 Lightsail 콘솔 터미널에 접속하여 `auto_deploy.sh`를 실행하는 것을 배포 표준 절차로 확립.

### 2. 다중/중복 로그인 보안 강화 (`login.js`, `login.html`)
*   **이슈:** 한 대의 PC에서 동시에 여러 탭을 열어 복수의 사원 아이디로 로그인(세션 덮어쓰기)이 가능했던 보안 취약점 존재.
*   **해결:** 이미 `localStorage`에 로그인된 상태(`isLoggedIn === 'true'`)라면, `login.html` 진입 시 알림을 띄우고 메인 대시보드 화면(`index.html`)으로 강제 이동시켜 중복 로그인을 차단함. 코딩 후 캐시 무력화를 위해 스크립트 파라미터(`v=2`) 업데이트 및 서버 배포 완료.

### 3. 인사기록카드 권한 복원 버그 수정 (`hr.js`, `hr.html`)
*   **이슈:** 신규 사원 등록 시 설정한 "세부 모듈별 권한(열람/수정)" 데이터가 수정창(수정 모달)을 열었을 때 체크박스에 복원되지 않던 버그 발생.
*   **해결:** `editEmployee` 함수 내에 사원 객체의 `permissions` 배열 인덱스를 읽어서 `.perm-table`에 있는 체크박스 요소를 매핑하여 복원해주는 로직을 추가. 스크립트 버전(`v=102`) 갱신 및 배포 완료.

### 4. 세부 모듈별 열람 권한 제어 실질화 및 서버 무한 루프 픽스 (`app.js`)
*   **이슈 1:** 029 사원에게 "HR/근태관리" 권한을 해제했으나, 사이드바 메뉴가 계속 노출되는 문제 발생.
*   **이슈 2 (치명적 오류):** 권한 처리 코딩 중, 권한이 없는 사용자를 단순 `login.html`로 돌려보내면 `login.js`의 중복 로그인 차단 로직과 맞물려 무한 리다이렉트가 일어나며 서버(브라우저)가 다운되는 블로킹 현상 발생. 
*   **해결:** 
    *   권한이 막힌 모듈은 `.nav-item` 디스플레이를 즉시 `none`으로 처리하여 메뉴 목록에서 완벽히 숨김.
    *   URL 수동 입력을 통한 직접 접근 시 경고 모달을 띄운 뒤, 강제로 로그아웃 처리(`localStorage.removeItem`)를 시켜 무한 루프 탁구 현상을 근본적으로 차단함.
    *   캐시를 뚫고 업데이트되도록 주요 html 파일들의 `app.js` 버전을 강제 갱신함.
