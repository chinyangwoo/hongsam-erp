홍삼스파 ERP - POS 매출 연동 기술 및 구현 가이드
ERP 통합 대시보드에 표시되는 금일 매출 및 월간 누적 매출을 POS(판매시점 정보관리 시스템) 와 실시간으로 연동하기 위한 백엔드 및 프론트엔드 통합 구성안입니다. 사용하는 POS 기종(OKPOS, KIS정보통신 등)에 따라 세부 스펙은 다를 수 있으나, 표준적인 연동 로직 단계는 동일합니다.

1. POS 연동의 3가지 핵심 방식
매장 내 POS에서 데이터를 빼내어 ERP가 있는 AWS 서버로 가져오려면 보통 다음 중 한 가지 방식을 채택합니다.

A. POS 벤더사 제공 Open API 방식 (가장 권장)
설명: POS 회사에서 제공하는 정식 API 서버에 우리 사장님 계정(토큰)으로 요청을 보내어 데이터를 받아오는 방식입니다.
장점: 보안이 뛰어나고, 서버 구성이 깔끔합니다.
B. 로컬 데이터베이스 직접 연결 (Agent 방식)
설명: 매장에 있는 메인 POS 데스크탑(주로 Windows)은 내부에 MSSQL이나 MySQL을 돌리고 있습니다. 포스기에 작은 프로그램(Agent)을 심어, 30분마다 매출 통계를 AWS 서버로 전송해주게 만듭니다.
장점: 벤더사가 API를 지원하지 않는 구형 POS라도 연동 가능합니다.
2. 시스템 아키텍처 (동작 순서)
[사용자] ERP 대시보드(웹) 접속
[ERP 화면(app.js)] AWS 서버에 GET /api/revenue?date=today 데이터 요청
[AWS 서버(Node.js 등)] POS API 서버로 사장님 인증키를 담아 매출 내역 질의
[POS API 서버] "오늘 매출 XX원, 이번 달 매출 XX원" 값 반환
[AWS 서버] 받은 데이터를 정제하여 ERP 화면으로 전달
[ERP 화면] 넘겨받은 텍스트를 대시보드 숫자에 렌더링
3. 프론트엔드 (app.js) 개발 구현방안
방금 업데이트 해드린 app.js 내부의 updateKPIs() 함수에 다음과 같이 비동기 호출 방식을 장착합니다.

javascript
// app.js 내 통신 로직 샘플
async function updateKPIs() {
    try {
        // 1. AWS 서버의 백엔드 API 라우터에 금일 및 월간 매출을 요청합니다.
        const response = await fetch('http://52.79.239.134:3000/api/revenue/summary', {
            method: 'GET',
            headers: {
                // 필요시 보안 토큰 명시
                'Authorization': 'Bearer ' + localStorage.getItem('erp_token')
            }
        });
        if (!response.ok) throw new Error('서버 네트워크 오류');
        
        const data = await response.json();
        /* 
           받아올 데이터 예시 (data 객체):
           {
               "todayRevenue": 1500000,
               "monthRevenue": 47000000
           }
        */
        // 2. HTML 태그에 금액 꽂아 넣기
        const kpiTodayRev = document.getElementById('kpi-today-rev');
        const kpiMonthRev = document.getElementById('kpi-month-rev');
        // toLocaleString()을 사용해 1,500,000 형태로 콤마(,) 포맷팅
        if (kpiTodayRev) kpiTodayRev.innerText = data.todayRevenue.toLocaleString();
        if (kpiMonthRev) kpiMonthRev.innerText = data.monthRevenue.toLocaleString();
    } catch (error) {
        console.error('POS 연동 에러:', error);
        // 에러 발생 시 처리 방안 (예: 임시로 0원 표기 또는 '연결 오류' 텍스트)
        document.getElementById('kpi-today-rev').innerText = "연결 오류";
    }
}
4. 백엔드 (서버) 개발 구현방안
Node.js(Express) 환경을 가정했을 때, POS 벤더가 제공하는 토큰 및 API 주소를 이용하여 요청을 중계해 주는 로직입니다. (CORS 문제 방지 및 API Secret Key 보안을 위함)

javascript
// server.js (Node.js 백엔드 예시)
const express = require('express');
const axios = require('axios');
const app = express();
app.get('/api/revenue/summary', async (req, res) => {
    try {
        // POS사 (예: OKPOS) 연동 API의 토큰 및 상점 코드
        const POS_API_URL = 'https://api.posvendor.com/v1/sales/summary';
        const STORE_CODE = 'HONG_SPA_001';
        const SECRET_KEY = '발급받은_POS_연동_시크릿키';
        // 1) POS 서버에 통신하여 매출데이터 콜
        const posResponse = await axios.get(`${POS_API_URL}?storeCd=${STORE_CODE}`, {
            headers: { 'Authorization': `Bearer ${SECRET_KEY}` }
        });
        // 2) 필요한 결과값만 정제 (예제)
        const todaySales = posResponse.data.total_today;
        const monthSales = posResponse.data.total_month;
        // 3) ERP 화면(app.js) 쪽으로 JSON 데이터 응답
        res.json({
            todayRevenue: todaySales,
            monthRevenue: monthSales
        });
    } catch (error) {
        console.error(error);
        res.status(500).json({ error: 'Failed to fetch from POS vendor' });
    }
});
// AWS 내부에서 서버 포트 구동
app.listen(3000, () => {
    console.log('ERP API Server running on port 3000');
});
